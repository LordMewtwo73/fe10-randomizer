Instructions

About Instructions How it Works FAQ Appendix

Looking for a step-by-step process to make this randomizer go? You've come to the right place!

  1. Find an NTSC-U or PAL FE10 iso. I can't tell you where to find this, but the internet is your friend here. Once you have it, make sure it is a .iso file, not .nkit.iso or .wpf or anything else.

  2. Download and extract the randomizer. You can find it on the DISCORD or on GITHUB.

  3. Extract the contents of your iso into a directory using a program like WiiBaFu. If you have problems with WiiBaFu, and you're sure you have a .iso, check out the FAQ to see if the problem has been covered there.

  4. Open up the randomizer by running the .exe file in your downloaded randomizer folder.

  5. After extracting with WiiBaFu, you should have a collection of folders that contain all of the data of your iso. Inside of the extracted folder, there will be a folder named "DATA", and then within that folder there should be a folder named "files". (Note: in some versions of the game, there is no "DATA" folder. However, you should still have a "files" folder. Using the Load Folder button under File on the randomizer menu bar, select the "files" folder. EXAMPLE: after extracting my iso to my desktop, I would select the following folder with the randomizer: C:\Users\LordMewtwo73\Desktop\Fire Emblem Radiant Dawn [RFEE01]\DATA\files

  6. Choose whatever settings you would like to modify with the randomizer. For a detailed explanation of each selection, check out the How it Works section of this document.

  7. Here you may choose to fully save your changes or not. If you select the checkbox named Don't Save, the randomizer will do all of its fancy math and calculations and spit out an outputlog for you to peruse. However, your ISO will be unchanged. This allows you to check the outputlog to make sure you like the final product before actually editing your files. If you use the same exact seed and settings (I recommend just not closing the randomizer), but you uncheck Don't Save, you will find that your randomized ISO will be identical to what your outputlog theorized. Whether you select that checkbox or not, click Randomize to begin the process!

  8. If the previous steps were done correctly, the randomizer should run without issue and eventually say "Randomization complete!"

  9. Use WiiBaFu to rebuild your iso from the extracted contents. Once again, if you come across problems with WiiBaFu, check out the FAQ.

  10. Congratulations, you have a randomized FE10 iso! A new randomization will require a new save file, but you can start from a cleared save file just fine. Make sure your new game is either Normal or Hard mode, as the randomizer does not work with Easy mode!

Classes

Class Randomization Options

Basic randomization keeps everybody as the same race even after class randomization. Using this function would prevent Sothe from becoming a Wolf, while simultaneously preventing Volug from being a Rogue.
When paired with random recruitment, the unit slot is instead kept the same race. (Example: Volug is now the unit recruited in 1-2 instead of Sothe. Even though Volug is usually a laguz, he will be a beorc class due to the fact that the original character recruited there is a beorc unit.)

Full randomization removes the restrictions of its predecessor, and allows for a no-holds-barred class randomization.

The rest of this dropdown contains a list of pre-weighted randomization "challenges" that usually restrict you to one or two types of classes.
Basic and Full allow you to customize the weights, which will be explained more in the next section.

At default, this option randomizes 69 units. It does not randomize the three herons without a different checkbox being selected, and it does not randomize the playable BK.

Weights

Class weights determine the probability of each character being randomized into that type of class. Each tier, from None to More Likely, increase the probability by a factor of 3 above the previous tier. We will discuss the calculations behind the program, and then detail what classes are contained in each classtype.

Each classtype is weighted by the equation W = 3x-1, where x equals the column number from 1(Less Likely) to 5(More Likely) (None sets W = 0). The program adds up all of the weights of each classtype to get a total, and then selects a random number from that total. Let's look at an example:

In this example, Infantry is set to the 4th column, Mages and Cavalry are set to the 2nd column, Beasts is set to the 3rd column, and everything else is set to zero. We can calculate the weight of each type using the equation above:

  • Infantry: W = 34-1 = 27
  • Mages: W = 32-1 = 3
  • Cavalry: W = 32-1 = 3
  • Beasts: W = 33-1 = 9
  • Everything else: W = 0

The total weight is Wtot = 27+3+3+9 = 42.

Dividing each weight by the total will give the percentage chance of selection:

  • Infantry: 27/42 = 64.3%
  • Mages: 3/42 = 7.1%
  • Cavalry: 3/42 = 7.1%
  • Beasts: 9/42 = 21.4%

With the 72-unit cast that is Radiant Dawn, on average you would have 46 infantry, 5-6 mages, 5-6 cavalry, and 15-16 beast units.

Exceptions

There are a few exceptions to randomized classes, most of which have been implemented to prevent unwinnable situations, or "softlocks".

  • Ike and Ranulf cannot be mounted due to "The General's Hand"*
  • Ike cannot be a magic unit due to his solo fight with BK**
  • Ike also can't be a raven or rogue due to the STR cap being too low to kill BK**
  • Ike, Micaiah, Sothe, Nephenee, and Brom cannot be herons
  • Micaiah and Edward cannot both be priests at the same time
  • There are a maximum of three herons per game

When a user selects options that would force one of these situations, the unit instead will default to their vanilla class.

*Ike and Ranulf can be mounted classes with the Horse Parkour setting under Miscellaneous.
**Ike can be magic/rogue/raven classes with the Nerf BK or Ike Fights BK with the Power of Friendship settings under Miscellaneous.

Class Base Stats

Class bases can be randomized in the same ways that personal bases can, so I won't go into detail again. If you haven't read the personal base stat section yet, it's right above this section.

The only difference is that the LCK stat for classes is always set to zero, so the shuffle option completely disregards luck entirely.

Set Stat Caps

This selection will change the stat caps of each class in the game, allowing player (and enemy) units to grow well past where the original game intended. The downside of higher caps is removing the ability to BEXP abuse, due to the rarity of a unit capping any of their stats.

No matter what you select here, the HP cap will be unshackled to 127 for all classes.

Random w/ Deviation

Instead of setting all stat caps to a selected value, this option lets you randomize stat caps similarly to how you would randomize growths or bases. Each stat cap is randomized to a value between its vanilla value +/- the selected deviation.

Flat Increase

This option will scale all stat caps up by the selected value. You should use this if you want to unshackle stats while also keeping each class's unique stat spread.

Only Change T3 Caps

Pretty self-explanatory. Use this option with any of the three Stat Cap modifiers and only the final tiered classes will be affected. This includes transformed laguz.

Minimum Cap

Forces each random cap to be equal or greater than the specified value here.

Randomize Stat Boosters

Enemies

Randomize Generic Enemy/Ally Classes

These selections will change the classes of all red or yellow units in the game. Each unit will be individually randomized, unless you select the "1:1 Class Changing" option.

Due to AI issues among other things, there are limitations:

  • Thieves are unchanged to keep confusion to a minimum, as they are able to open chests regardless of class
  • Some flying units are unchanged if they spawn in impossible-to-reach locations (ie pegasus knights in 1-6)
  • Tier1 units cannot change into laguz because they become nearly unbeatable
  • Due to their AI not knowing how to transform, all beorc turned into laguz classes are considered feral ones, and will always stay transformed - As of version 3.0.0, this is no longer the case
  • Allied laguz remain unchanged - As of version 3.0.0, this is no longer the case

1:1 Class Changing

This selection will 1-to-1 change classes of enemy units on all chapters. By 1-to-1, I mean that every fighter on a chapter will randomize into the same class, and so forth. Two units who are originally the same class as each other will not randomize into two different classes. This does not include units on different chapters. For example, bandits in 1-1 may change into myrmidons, but bandits in 1-2 may become pegasus knights.

Include Spirits

This selection not only randomizes the spirits that you fight in the final chapters of the game, but also adds the three spirit classes to the list of possible classes the enemy can turn into in part 4.

Similar Classtype

Makes sure that the random enemies stay within the bounds of their vanilla class. For example, an enemy myrmidon may randomize into a soldier,fighter,archer,etc, while an enemy firemage may randomize into a windmage,lightmage,etc. The "types" that this selection follows are Infantry, Magic, Mounted, Armored, Flying(beorc), and Laguz

No Siege Tomes

Any enemy unit randomized into a magic class will be prevented from having meteor/bolting/etc.

Only Siege Tomes

Conversely, any enemy unit randomized into a magic class will always have meteor/bolting/etc. Since early siege tomes can create impossible scenarios, this change does not begin until part 3.

Don't Randomize Healers

Enemy and allied priests and bishops will not be modified in any way. Tier 1 units cannot be randomized into the priest class, and tier 2 units who are randomized into a bishop will be equipped with light magic instead of a staff.

Beorc can't be Laguz and Vice-Versa

There are some AI issues when Laguz enemies are changed into Beorc classes and vice-versa. This selection keeps random enemies within their default race, keeping a more balanced and intelligent enemy force than with full randomization.

Random Enemy Drops

Increase Enemy Growths

In case you were not aware, enemies do not have personal base stats. Instead, the game takes their class bases, treating them as level 1 stats, and autolevels them up using averaged growths. This selection will increase those growths, which will increase the stats of higher-leveled enemies.

Randomize Boss Classes

This will randomize the class and weaponry of almost every boss in the game. The list of possible weapons is the same for generic enemies, but as a default bosses are considered to have the "stronger weapons" checkbox selected, so their weaponry can be one tier higher than generic enemies. Limitations include:

  • Ike, Lethe, Micaiah, and Sigrun are not randomized as bosses
  • Laguz bosses are not randomized
  • bosses cannot be randomized into laguz or spirit classes
  • BK, Dheginsea, and Ashera are not randomized

Randomize Laguz Weapons

Without this setting, random weapons will only modify beorc weapons - swords, lances, etc. Laguz weapons are tied to two things, their class and their weapon level. For each laguz class, there are three weapons (A, S, and SS rank). For example, all tigers with A rank in strike will have the same weapon stats, regardless of who they are. If you randomize these weapons, there is no guarantee that the SS rank weapon will be stronger than the S or A rank, so this selection can be very risky.

With this selection, staves with a low number of uses (Sleep, Silence, Matrona, etc) will not have their number of uses modified when weapons are randomized. This can be helpful to prevent enemies from having 15-use Sleep staves and similar situations.

Similar to the previous selection, this will prevent seige tomes and ballistae from having their number of uses modified. Though a 22 use Meteor tome sounds great, it can be a lot less fun when the enemy has them, too!

Characters

To understand random recruitment, it is important to make the distinction between a unit (Nolan) and a unit slot (unit you obtain in 1-1). When you randomize recruitment, some of the unit information will be taken from the new unit, while some will stay consistent to the slot, no matter who takes up that slot.

The following are characteristics taken from the new unit:

  • Name
  • Face
  • Description
  • Affinity
  • Growths
  • Class (if otherwise unrandomized)

The following are characteristics consistent with the unit slot:

  • Level
  • Skills (if otherwise unrandomized)
  • Events (win/lose conditions, killing Ashera, seizing, finding treasure, PRF weapons)
  • Bases
  • Special battle music

Thus, the unit that replaces Ike is able to wield Ettard and Ragnell (if their class allows swords), and more importantly, must be the character to land the finishing blow on Ashera. Similarly, the unit that replaces Sothe will have an increased chance to find hidden items on the map, an innate ability normally set to Sothe and Heather.

As of version 2.2.2, bases of the slot are reordered to better reflect the new character. For example, Ike's highest personal base is in STR. If he were to be randomized into Soren, whose highest personal base is MAG, then Ike's slot will have a MAG base equal to vanilla Ike's STR base stat.

Skills

Randomize Skills

Randomizes each skill attached to playable characters (sans BK). The number of skills will depend on the user inputs. The list of possible skills can be found in the appendix of this document.

If a unit obtains a skill normally locked to the character it starts on (ie Vigilance, Glare), it will be locked to the new unit.

If a unit obtains a skill not normally allowed to be on their class (ie Blood Tide on Soldier), it will still work. However, if you remove the skill, you will not be able to reassign it to that unit.

Characters keep same # of Skills + modifier

Each character will gain a number of random skills equal to the number of skills they start with in the vanilla game, with the addition of whatever number you put in the modifier box. For example, Micaiah normally starts with 1 skill, but Ena starts with 4. If you select this option with a modifier of 1, Micaiah will start with 2 random skills and Ena 5. Each character can have a maximum of 6 skills.

Characters have set # of Skills = modifier

No matter how many skills a character may normally have in the base game, this option will give all player characters the same number of skills, equal to the number you place in the modifier box.

Give all Characters/Laguz Paragon

Replaces the final skill of each character with Paragon. If a character has less than the maximum number of skills, Paragon will not overwrite any of their original or randomized skills. You can also limit this to modify Laguz characters only.

All Dragons can use Pool Skills

Blood tide, white pool, night wave can all be equipped to a dragon regardless of color with this setting.

All Classes can use all Skills

This setting removes all restrictions for skill placement. Beorc can have Laguz skills, mages can have Imbue, etc.

Randomize Skill Capacity

This modifies the necessary capacity of all skills in the game, including the mastery skills of T3 classes, like Astra. Each skill will have a chance to stay unmodified, a large chance to be +/- 5 capacity, a slighly less chance of being +/- 10 capacity, and a very small chance of being +/- 15 capacity.

Give All Enemies Skills

This option will give all enemies, generic and bosses alike, random skills up to the selected number. The maximum number of skills for enemies is three.

Give Bosses 1 More Skill

Does exactly what it says. If you selected two for enemy skills, this option will give bosses three skills. If you select this without selecting the enemy skill option, this will give each boss one random skill.

Shop

Randomize Shops

This selection changes the contents of the armory/shop in each chapter that normally has one. The possible items include weapons, skills, and stat boosters.

Healing items (ie Herbs), promotion items, and healing staves are twice as likely to appear as other items.

Force Basic Items at Each Shop

This selection will force all shops to contain iron weapons, herbs, olivi grass, and heal staves

Randomize Bargains

This will randomize the bargain selection of each shop, where each item is a single-purchase item. The weights allow you to better customize what you may see in the bargain section. A higher number means that it is more likely that an item of that type is in the bargains. Setting a value to zero will insure that you will never see an item of that type in the bargains.

I've gotta admit, I'm pretty proud of this one. It took me forever to figure out how to do it, but now each weapon in the forge can be randomized, and they change each chapter as well. Under "Iron Sword", there can be a Windsword, or a Brave Axe, or even Thani! Some clarifications:

  • This does not change which chapters have a forge.
  • This does not modify the number of forging items there are per chapter. As normal, Part 1 will have less forging options than part 4.
  • The weapon models are based off of items that can normally be forged, so a forged Killing Edge may look like a Steel Sword.

Just like the bargain section, this option has weights to allow you to further customize what you may find in the forge. A higher number means that it is more likely that a weapon of that type can be found in the forge. Setting a value to zero will insure that you will never see that weapon in the forge.

Randomize Movement

A long-asked for addition to the randomizer, now you can randomize the amount of movement each class has. Each class will be assigned a random movement between the selected minimum and maximum values. In order to prevent softlocks due to tiles such as swamp and ledges that persist throughout the game, this option will also reduce the movement cost of such tiles to be equal to the selected minimum.

For example, if you were to select a minimum movement of 3, the cost for thieves to cross a swamp will be unchanged, as it is normally 2, but the cost for an armored unit to cross a swamp, normally 4, will be reduced to 3.

This will not change movement costs that are set to be impassible, such as mounted units unable to climb ledges. However, if this option is selected alongside Horse Parkour, the movement cost will be lowered from 6 to the selected minimum.

Randomize Promotion

This selection will randomize the promotion path of each class that is able to promote. Druids will still require a Master Crown to promote. Classes that are usually not in promotional paths, ie Queen, are able to become part of a promotional path. No two classes are able to promote into the same class.

Force Physical->Physical, Magic->Magic

For anybody desiring a less intense random promotion. This selection will make sure that promotion paths will keep physical and magical classes separate, so you don't have armor knights promoting into bishops.

FAQ

About Instructions How it Works FAQ Appendix

How do I promote Ike/Micaiah/Sothe?

All event promotions are turned off. Now these three promote like any other unit, either with a Master Seal/Crown or at level 21.


My druid capped at level 21! Isn't he supposed to promote?

Due to the druid being an enemy-only class, the game does not allow it to promote using exp or bexp. You must use a Master Crown to promote to summoner.


If I randomize classes and Ike can't use Ragnell, how do I defeat Ashera?

Ike (or whoever replaces him with random recruitment) must land the final blow on Ashera, regardless of (blessed) weapon.


WiiBaFu is throwing an error when I try to extract my rom!

Make sure your copy of Radiant Dawn is in .iso format. If it is a .iso.nkit, you can follow instructions HERE and in the comments of that youtube video to convert it into a .iso, or you could just find a .iso formatted copy online if that is too much work.

If your copy of Radiant Dawn is a .iso and you still get this error, you may be trying to extract your ISO to a location that already has an extracted ISO. WiiBaFu seems to not like that, but it doesn't do a great job of informing the user.


Why can't my units be Bandits?

For some reason, Radiant Dawn removed the Berserker class, so Bandits do not have a promotion path. I could probably make them promote into Warriors, but at the end of the day all that will do is double the chances that the Dawn Brigade is filled with axe users.


The randomizer produces an error that tells me that it can't find assets/CharacterInfo.csv, but it's in the correct folder!

Make sure that you did not move/rename any files in the randomizer. If everything is exactly as it was when you unzipped it, try running the .exe as administrator.


The randomizer produces an error that says "Cannot find file [something something]/temp/script/[name].cmb"

This is a known issue with the script editor; make sure the filepath does not contain any non-English characters. It also would be best if the path is on the C: drive, not an external drive.


Where do I get an ISO?

I cannot tell you how to find this, other than the internet cue Timmy Turner. Please do not ask for this on the Discord, either.


I keep getting the error message "Cannot find DATA/sys/main.dol"

Make sure that the folder you're selecting with the randomizer is DATA/files.


Do the randomized games run on console/mobile?

There have been varying reports of people having no problems, and other people having unskippable crashes.


Why are some of my units T-posing?

A few units on specific chapters attempt to use their default models for some reason. I am not sure why, all I know is that if a unit dies while T-posing, it'll crash the game. So if you notice T-posing, it may be in your best interest to set the animations to map-only for the rest of that chapter.


I selected Ironman mode but Aran dying still caused a game over, what's up?

This is a known glitch that I am unable to fix. After being recruited on his debut chapter (1-3), Aran dying will result in a game over. If he dies as a red unit, this does not count as a game over. This is the only game over that is not due to a victory condition/lord dying. For a full list of game overs in Ironman mode, check out the README under "Ironman mode"


How to I check my growths after randomizing?

There is a new file in the randomizer folder called "outputlog.htm" that appears after a successful randomization!
